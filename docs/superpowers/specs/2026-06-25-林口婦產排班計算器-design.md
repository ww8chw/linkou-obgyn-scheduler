# 林口長庚婦產科住院醫師排班計算器 — 設計文件

日期：2026-06-25
形式：單一 HTML 檔（純前端 HTML+CSS+JS，可離線、可用檔案分享）

---

## 1. 目的與範圍

每月計算林口長庚婦產科住院醫師「各班別、各職級、每人應值幾平幾假」的**數量分配**工具。

- **只算數量**，不產生逐日班表（哪天誰值由排班者自行安排）。
- 排班者每月打開網頁、輸入當月參數、按計算、看結果。
- 「病房班（最後一天）」與「L5（扣童綜合外訓後）」的剩餘需求，由排班者自己先扣外訓後直接輸入數字（不在系統內處理外訓人員的明細）。

### 範圍外（不做）
- 不排逐日班表、不指定特定日期。
- 不處理外訓人員自身的明細班數（output 不顯示外訓）。
- 平常的病房班（PGY1 值、隨意排）不納入。

---

## 2. 名詞與班別定義

每天都要各有 1 人值的固定班別（總需求 = 當月對應平/假天數）：

| 班別 | 說明 | 可值職級順位 |
|------|------|------|
| L5 | — | Y2、R1（扣病房班後）→ R2 → R3 → R4 → F1 |
| L1 | — | R2 → R3 |
| L | 總值班 | R3（受限）→ R4 → F1 → F2 |
| T1 | 台北班 | 台北R1-3 → 台北R4 → 林R1 → 林R2 → 林R3 → 林R4 → 台北F1 |
| T2 | 台北班 | 台北F1 → 林R4 → 林F1 → 林F2 → 林F3 |
| LR | 待產室 | 林R4 → 林F1 → 林F2 → 林F3 |

特殊需求：

| 班別 | 說明 | 可值順位 |
|------|------|------|
| 病房班 | **只有月底最後一天**有洞（PGY1 不能值），共 input 指定的平/假洞數 | 家醫外訓 → 急診外訓 → 童綜合外訓 → Y2(可值最後一天者) → R1 → R2 → R3 |

> 註：input 已要求排班者先扣掉外訓能值掉的病房班與 L5，故系統實際從 Y2/R1 起算病房班、從 Y2/R1 起算 L5。外訓不在系統內處理。

職級高低：`Y2 = R1 < R2 < R3 < R4 < F1 < F2 < F3`

## 3. 輸入（Input）

### 3.1 月份與天數
- **選擇年月**：2026 年（先只支援 2026）。
- 系統依內建 2026 台灣官方行事曆（行政院人事行政總處公告，含國定假日、連假、補班日）自動算出：
  - 當月**平日班天數**、**假日班天數**（補班的週六算平日、連假內的平日算假日）。
  - 當月**最後一天是平日還是假日**。
- 三個欄位皆**可手動覆寫**（保險用）。

> 實作注意：2026 國定假日/補班日需依官方公告逐日核對後寫死於程式中。系統僅保證已寫入年份（2026）正確。

### 3.2 台北人員（值 T1/T2）
- 台北 R1/R2/R3：_人（值 T1）
- 台北 R4：_人（值 T1 或 T2）
- 台北 F1：_人（值 T1 或 T2）
- 台北人員每人比照林口套用配額上限（見 §5），照順位填，填不滿才往下給林口。

### 3.3 林口人員（已扣產房班）
各職級人數，以及細項：

- **Y2**：_人，其中 _人可月底最後一天值班
- **R1**：_人（大特休 _人、小特休 _人）
- **R2**：_人（大特休 _、小特休 _、基隆籍 _）
- **R3**：_人（大特休 _、小特休 _、基隆籍 _、**可值總值班人數 _、共可值總值班班數 _**）
- **R4**：_人（大特休 _、小特休 _、基隆籍 _）
- **F1**：_人（大特休 _、小特休 _）
- **F2**：_人（大特休 _、小特休 _）
- **F3**：_人（大特休 _、小特休 _）

> 基隆籍：不能值 T1/T2。
> R3「可值總值班人數/班數」：本月 R3 只有這幾人可值 L，且合計不超過該班數上限。R4 全部可值 L、無上限。

### 3.4 外訓扣除後的剩餘需求（排班者自行先扣外訓）
- **病房班**（月底最後一天，扣外訓後剩餘）：_平 _假
- **L5**（扣童綜合外訓後剩餘）：_平 _假

> 其餘班別（L1、L、T1、T2、LR）總需求 = 當月平/假天數，系統自動帶入。

## 4. 演算法

### 4.1 整體流程
方法為 **(A) 先按班別順位灌滿，再檢查/微調**。

**班別計算先後順序**（越受限者越早分，先卡位）：

1. 病房班（最後一天）
2. L5
3. L1
4. L
5. T1
6. T2
7. LR

每個班別依其「職級順位」逐級填補：把該班別的平日需求、假日需求**分開**，由順位最前的職級開始吃（受該職級每人配額上限約束），吃不完才往下一順位職級。同職級內部依公平原則均分。

### 4.2 各班別填補細節
- **病房班**：平/假各自，依順位 Y2(可值最後一天者)→R1→R2→R3 填，受人數限制。
- **L5**：Y2、R1 先扣掉已背的病房班後，剩餘配額填 L5；不夠往 R2→R3→R4→F1。
- **L1**：R2 優先，不夠 R3。
- **L**：先填 R3（受其「可值人數」與「班數上限」雙重限制），再 R4（全可值、無上限），不夠 F1，再不夠 F2。
- **T1**：台北R1-3 → 台北R4 → 林R1 → 林R2 → 林R3 → 林R4 → 台北F1。基隆籍跳過。
- **T2**：台北F1 → 林R4 → 林F1 → 林F2 → 林F3。基隆籍跳過。
- **LR**：林R4 → 林F1 → 林F2 → 林F3。

### 4.3 平假分配與職級單調性
- 平日、假日各自獨立按順位填。
- 假日班需維持**職級單調性但差距適度**：低職級假日班數 ≥ 高職級，但避免極端差距（不得出現如「R2 三個假日、R3 零個假日」）。實作上：分配假日時盡量讓相鄰職級差距最小化，同時不違反順位。

### 4.4 每人配額與特休
- **(a) 每人基準**：某職級分到的該班別/總班數，**除以人數**盡量平均（平日均分、假日均分），自然落在上限內。無另設目標值。
- **(b) 特休扣班**：先算該職級**無特休者**每人 N 平 M 假；
  - 大特休者 = (N−2) 平、M 假
  - 小特休者 = (N−1) 平、M 假
  - **假日班不因特休而減，只減平日**。
- **(c) 轉嫁**：特休者少值的平日班由**同職級其他人吸收**（其他人平日 +）。若吸收後同職級有人 > 8 班，則該超出部分**往上一個職級順位推**（轉給上一順位職級承接）。

### 4.5 最低/最高值班限制
- **硬性上限**（絕對不可破）：每人 ≤ 8 班、≤ 3 假。`5平3假` 與 `6平2假` 皆可接受。
- **F 級最低值班**：F1 ≥ 3 班、F2 ≥ 2 班、F3 ≥ 1 班。
- **R1–R4 下限**：除非有 F1/F2/F3 未達最低值班數的情況，否則 R1–R4 不得低於 8 班。

### 4.6 規則優先順序（衝突時的取捨）
由高到低：

1. **硬性上限**：≤8 班、≤3 假（5平3假 / 6平2假皆可）。**絕對不破。**
2. **班別順位**：各班別職級填補順位 + 基隆籍不值T1/T2。**絕對遵守。**
3. **特休扣班**：大特休 −2 平、小特休 −1 平。
4. **職級單調性**：低職級總班數 ≥ 高職級、假日班 ≥ 高職級（適度、台北輪訓除外）。
5. **組內公平**：同職級假日盡量等量、班數盡量均分（除不盡允許差 1）。

## 5. 輸出（Output）

不顯示外訓明細。依下列格式呈現各職級結果：

**台北輪訓**
- R1-3 _人：總共應值 T1 _平_假
- R4 _人：總共應值 T1 _平_假、T2 _平_假
- F1 _人：總共應值 T1 _平_假、T2 _平_假

**林口各職級**（每個職級顯示：各班別總數 → 每人平假 → 大/小特休者平假）

- **Y2** _人：病房班 _平_假、L5 _平_假 → 每人 _平_假
- **R1** _人：病房班 _平_假、L5 _平_假、T1 _平_假 → 每人 _平_假；大特休(_人)_平_假；小特休(_人)_平_假
- **R2** _人：L1 _平_假、T1 _平_假 → 每人 _平_假；大特休…；小特休…
- **R3** _人：L _平_假、L1 _平_假、T1 _平_假、L5 _平_假 → 每人…；大/小特休…
- **R4** _人：L _平_假、L1 _平_假、T1 _平_假、L5 _平_假、T2 _平_假 → 每人…；大/小特休…
- **F1** _人：L _平_假、L5 _平_假、T2 _平_假、LR _平_假 → 每人…；大/小特休…
- **F2** _人：T2 _平_假、LR _平_假 → 每人…；大/小特休…
- **F3** _人：T2 _平_假、LR _平_假 → 每人…；大/小特休…

### 5.1 警示
若無法滿足硬性規則（例如人力不足導致有人必須 > 8 班、或 F 級未達最低、或某班別湊不滿），output 以明顯警示列出**哪個班別/職級超載或缺人**，讓排班者人工調整，而非靜默產生錯誤結果。

## 6. 介面與技術

- 單一 `index.html`，內含 CSS + JS，無外部相依。
- 版面：上方輸入區（分組：月份/台北/林口各職級/外訓剩餘需求），下方「計算」按鈕，再下方結果區。
- 內建 2026 台灣官方假日資料（陣列/物件），用於自動算平假天數與最後一天屬性。
- 計算純前端即時完成，無後端、無網路需求。

## 7. 已確認的決策摘要

- 病房班只有最後一天 2 洞，外訓由排班者先扣，系統只收剩餘平/假數。
- 台北人員比照林口配額（方法 A），T1/T2 每天各 1 人。
- 系統只算數量分配，不排逐日班表。
- 演算法：先按班別順位灌滿再微調；班別順序＝病房班→L5→L1→L→T1→T2→LR。
- 規則優先序：上限 > 班別順位 > 特休扣班 > 職級單調性 > 組內公平。
- 上限：≤8 班、≤3 假（5平3假 / 6平2假皆可）。
- L：R3（受人數+班數上限）先於 R4；R4 全可值無上限。
- 配額＝總數除人數均分；特休只減平日，假日不減；特休少值班同級吸收，超 8 則往上推。
- 假日班維持職級單調但差距適度。
- 單一 HTML 檔；月份自動帶入 2026 假日（可手動覆寫）。
